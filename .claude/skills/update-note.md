# 笔记补充技能

根据用户的 Java 代码改动，自动在对应笔记文件中补充实现说明。

## 触发条件

用户说"补充笔记"、"更新笔记"、"根据代码写笔记"或类似表述时触发。

## 执行流程

### 1. 获取代码改动

```bash
git diff -- <Java源文件路径>
```

只看 Java 代码的改动（`src/main/java/` 下的文件），忽略笔记文件本身的改动。

### 2. 定位笔记文件

笔记文件路径：`Redis学习笔记/Redis学习笔记/Redis学习.md`

先读取笔记末尾，确认当前写到了哪个章节，判断是否需要新增章节还是补充现有章节。

### 3. 分析代码改动

搞清楚三个问题：
- 这次改的是什么功能点？（缓存穿透、缓存更新、分布式锁…）
- 代码里加了什么关键逻辑？（新的判断分支、新的常量、新的注解…）
- 和之前相比流程变了哪里？

### 4. 写笔记

**风格要求：**
- 中文，口语化但技术准确
- 先贴关键代码（带注释），再列几个关键点（用 `**加粗**` 做小标题）
- 代码只贴改了的部分/核心方法，不用贴整个类
- 每个关键点一两句话说清楚即可，别写长文
- 章节编号跟着已有节奏走（第5节、第6节…）
- 跟前面笔记的格式保持一致

**示例格式：**

```markdown
#### N、xxx功能点

**简短描述做了什么**

```java
// 核心代码
```

**几个关键点：**

- **关键点1**：解释
- **关键点2**：解释
```

### 5. 确认和修正

补充完后，告诉用户写了什么。如果用户说有遗漏或不对，再针对性修改。
